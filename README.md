# Report - Happ Routing Fix ⚙️

Небольшой фикс для Happ на Windows 10, который чинит routing после реконнекта.  
* На Windows 11 не тестировалось, но, скорее всего, работает  
* Насчёт Win7/8.1 — я хз  
* Linux / macOS — нет, мне не на чем это тестировать  

---

## 🧩 Проблема

`sing-box`, движок на котором работает VPN, обновился и теперь ожидает `rule_set`.  
А Happ до сих пор может собирать конфиг со старыми геоданными вида `geosite:CATEGORY-RU`, `geosite:VK`, `geoip:RU`.

Из-за этого маршрутизация может работать неправильно (или не работать вообще).
Отсюда вытекает проблема — Российские сервисы часто сбоят или работают нестабильно на зарубежном IP.
С включённой маршрутизацией таких проблем не будет.

---

## 🛠️ Что делает фикс

Фикс следит за `config.json` Happ и после реконнекта подменяет старую схему `geosite/geoip` на локальные `rule_sets`.

Работает только с уже установленным и рабочим Happ и чинит именно routing-часть итогового конфига.

---

## 🚀 Как пользоваться

1. Запустите `happ-routing-fix.bat`
2. В консоли по дефолту должен быть статус **"ЗАПУЩЕНО"**, если нет — нажмите `[3]`
3. В Happ нажмите Connect и, очень желательно, переподключитесь ещё раз
4. Проверьте, работает ли роутинг.  
   РФ сервера должны быть с российского IP (проверить можно на [2ip.ru](https://2ip.ru)), зарубежные сервера — с VPN IP.  
   Банально посмотрите на треки с VK Музыки, фикс изначально под него затачивался
5. Когда в консоли статус **"ЗАПУЩЕНО"**, фикс следит за конфигом Happ в фоне

По дефолту “F2 Оставаться в фоне после закрытия” включён, поэтому можно просто закрыть консоль — он продолжит работать в фоне.  
Повторный запуск `happ-routing-fix.bat` откроет уже существующую консоль (если она ещё не запущена).

---

## ⚠️ Важно

Иногда Happ применяет изменения не с первого раза.

Если после 2-х переподключений фикс не работает:
- полностью выключите Happ, включая tray
- запустите его заново
- переподключитесь 1–2 раза

При таком случае выход из tray обязателен — нужно добиться полного перезапуска Happ.

---

## 🧠 Как работает

Фикс патчит файл:
`%LOCALAPPDATA%\Happ\config.json`

Он добавляет локальные rule_set:
- `geosite-vk`
- `geosite-category-ru`
- `geoip-ru`

И прописывает правила:
- VK / российские домены и IP → `direct`
- остальное → `proxy`

---

## ⚙️ Управление в консоли

- `[1]` обновить rulesets вручную  
- `[2]` открыть лог  
- `[3]` запустить / остановить фикс  
- `[4]` открыть чекер процессов  
- `[5]` Telegram разработчика  
- `F1` автозапуск Windows  
- `F2` работа в фоне  
- `F3` автообновление rulesets  
- `Esc` выход  

По умолчанию:
- автозапуск Windows выключен  
- работа в фоне включена  
- автообновление rulesets включено  

(рекомендуется включить автозапуск)

---

## 📌 Примечание

Happ может пересобирать `config.json` при реконнекте, поэтому фикс работает как watcher — он ловит изменения и повторно применяет правила.

Иногда нужен полный перезапуск Happ, если он игнорирует изменения.
