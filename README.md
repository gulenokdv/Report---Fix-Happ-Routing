# Report - Happ Routing Fix

Небольшой фикс для Happ на Windows 10, который чинит routing после реконнекта.
* На Windows 11 не тестировалось, но предположительно, должно работать. Насчет win7/8.1 я хз
* Linux/MacOS - нет. мне не на чем тестировать

Проблема: `sing-box`, движок на котором работает VPN, обновился и теперь ожидает `rule_set`, а Happ до сих пор может собирать конфиг со старыми геоданными вида `geosite:CATEGORY-RU`, `geosite:VK`, `geoip:RU`. Из-за этого маршрутизация может работать неправильно.

Этот фикс следит за `config.json` Happ и после реконнекта подменяет старую схему `geosite/geoip` на локальные `rulesets`.
Фикс рассчитан на уже установленный и рабочий Happ и чинит именно routing-часть итогового конфига.

## Как пользоваться

1. Запустите `happ-routing-fix.bat`.
2. В консоли по дефолту должен быть статус `ЗАПУЩЕНО`, если нет, запустите цифрой [3].
3. В Happ нажмите Connect и, очень желательно, переподключитесь ещё раз.
4. Проверьте, работает ли роутинг. РФ сервера должны быть с российского IP (проверить можно на [2ip.ru](https://2ip.ru)), зарубежные сервера — с VPN IP. Банально посмотрите на треки с VK Музыки, фикс изначально под него затачивался.
5. Когда в консоли статус `ЗАПУЩЕНО`, фикс следит за конфигом Happ в фоне.
По дефолту "F2 Оставаться в фоне после закрытия:" Включен, а значит, если вы ничего не меняли
* Можно просто закрыть консоль. Она будет работать фоном. Повторный запуск `happ-routing-fix.bat` раскроет уже существующую консоль (если ее еще нет).

Иногда Happ применяет изменения не с первого раза. Если после 2х переподключений фикс не работает, полностью выключите Happ, включая tray, запустите Happ заново и переподключитесь 1-2 раза.
* При таком случае, Happ из tray выключать ОБЯЗАТЕЛЬНО. Важно добиться полного отключения.

* * НИКАКИЕ правила маршрутизации создавать в Happ не требуется. Фикс пропатчит config.json по пути AppData/Local/Happ/config.json имитируя будто правила есть, т.е geodat качать не нужно.
(вы их и не скачаете без создания правила, но к слову.)


## Файлы

- `happ-routing-fix.bat` - основная консоль фикса.
- `happ-routing-process-check.bat` - чекер процессов для отладки. Можно быстро посмотреть, работает ли фикс, и при необходимости убить все найденные процессы фикса для полного перезапуска.
- `scripts/` - PowerShell-скрипты фикса.
- `rulesets/` - локальные rule-set файлы для sing-box. (замена старых geodat)

## Важно

Не удаляйте папку `rulesets`. Без файлов из этой папки фикс не сможет прописать рабочие `rule_set` в конфиг Happ.

Фикс использует локальные `rulesets` из папки проекта и может автоматически обновлять их из публичных upstream-репозиториев на GitHub.

Фикс хранит свои настройки в `happ-routing-settings.json`. 

## Управление в консоли

- `[1]` обновить rulesets вручную.
- `[2]` открыть лог.
- `[3]` запустить или остановить фикс.
- `[4]` открыть чекер процессов для отладки.
- `[5]` открыть Telegram разработчика.
- `F1` включить или выключить автозапуск Windows.
- `F2` оставаться в фоне после закрытия консоли.
- `F3` включить или выключить автообновление rulesets.
- `Esc` выйти из консоли.

По умолчанию:

- автозапуск Windows выключен;
- работа в фоне после закрытия консоли включена;
- автообновление rulesets включено.

(Рекомендуется включить автозапуск Windows)

## Что делает фикс

Фикс патчит файл: %LOCALAPPDATA%\Happ\config.json

Он добавляет локальные `rule_set`:

- `geosite-vk`
- `geosite-category-ru`
- `geoip-ru`

И прописывает правила: VK/российские домены и российские IP идут через `direct`, а всё остальное уходит через `proxy`.

## Примечание

Happ может пересобирать `config.json` при реконнекте. Поэтому фикс работает как watcher: он замечает перезапись конфига и снова применяет нужные правила. Но иногда бывает игнорирует наш фикс, по той причине требуется полный перезапуск именно самого Happ, и 1-2 reconnect.
